01

Design Philosophy

The Expanse UI is functional first, beautiful second. Every element serves a purpose. There is no decoration for its own sake. The team received guidance from a NASA astronaut on information hierarchy — what matters most when inches of steel separate you from vacuum.

Nothing was meant to be superfluous or just there for visual interest. Everything was a part of that UI for purpose. — Rhys Yorke, Motion Graphics Designer, Seasons 3 & 5

Core Principles

PrincipleImplementation
Operator-firstBuilt for crew under duress, not consumers browsing. Hierarchy must be instantly readable at a glance.
Front-end / back-end hybridCommand-line terminal text is visible alongside polished controls. This "under the hood" quality gives each ship personality.
Military disciplineClarity under stress. Fast decision-making. Minimal cognitive load. Deliberate color restrictions.
Practical fidelityScreens were real on set — designed at actual monitor dimensions, not idealized mockups. If it doesn't work at 14", it doesn't ship.
Faction identity in one frameViewer should know which ship they're on the moment a screen appears. Color, shape language, and texture all signal faction.

Faction Color Palettes

Each faction owns a color system. The designer's first goal was instant identification: red = Mars, blue = Earth, amber = Belt. Within each system, hierarchy is maintained through opacity, size, and shape — not additional hues.

MCRN / Rocinante — Mars

The primary reference ship. Red as the dominant color presents a unique challenge: error states and alerts must be distinguished from the already-red baseline. The solution is yellow borders for warnings and opacity reduction for inactive controls.

Typography System

The show uses a modified version of DIN Pro — a German industrial standard typeface. Geometric, supremely legible, functional. A fan-made recreation called Protomolecule is available on GitHub.

ElementStyleExample
Panel headersALL CAPS, wide tracking, small sizeletter-spacing: 0.15em; font-size: 10px
System readoutsMonospaced, tabular figuresfont-family: 'Overpass Mono', monospace
Values / numbersLarge weight, high contrastfont-weight: 700; color: #FFFFFF
AlertsLarge, bold, ALL CAPS — but proportionateNever screen-filling. Showrunner rejected that trope.
Terminal logMonospaced, dimmed (~70% opacity)color: rgba(180,180,185,0.7)
Secondary infoSentence case, muted colorcolor: var(--ex-text-dim)

Font Stack

/* Display / UI labels */
font-family: 'Protomolecule', 'DIN Pro', 'Overpass', 'Barlow', sans-serif;

/* Terminal / readout text */
font-family: 'Overpass Mono', 'Space Mono', 'IBM Plex Mono', monospace;

Terminal + GUI Hybrid

The Rocinante's signature trait: polished graphical controls sit alongside raw terminal output. Log data and diagnostic text scroll in monospaced type while the operator interacts with buttons and panels. This creates the "crew interface" personality.

Screen Layout Patterns

Cockpit Multi-Panel

Screens wrap around operator stations. Each screen has a dedicated function. Information is organized in bordered panels with clear headers.

┌──────────────────────────────────────────────────────────┐ │ ┌─── NAVIGATION ───┐ ┌─── WEAPONS ────┐ ┌─── COMMS ────┐ │ │ │ │ │ │ │ │ │ │ │ star map/track │ │ fire control │ │ channels │ │ │ │ + orbit lines │ │ + PDC status │ │ + messages │ │ │ │ + contact list │ │ + targeting │ │ + alerts │ │ │ └───────────────────┘ └────────────────┘ └──────────────┘ │ │ │ │ ┌─── SHIP SYSTEMS ───────────┐ ┌─── TERMINAL LOG ────────┐ │ │ │ reactor / life support │ │ [scrolling monospace] │ │ │ │ fuel / atmosphere / temp │ │ timestamped entries │ │ │ └────────────────────────────┘ └──────────────────────────┘ │ └──────────────────────────────────────────────────────────┘

Panel Construction Rules

  • Borders: 1px solid, faction primary at 40–60% opacity
  • Panel background: darker than screen background (layered depth)
  • Headers: ALL CAPS, small monospace, thin rule underneath
  • Content padding: generous — never cramped
  • Status bars: thin strips along bottom edges of panels
  • No rounded corners — sharp rectangles throughout (military/industrial)

UI Component Library

Buttons

Bordered rectangles, not filled. Border = faction primary at 45–60% opacity. Hover brightens border and adds faint fill. No rounded corners. No drop shadows.

Button states

Status Indicators

Status dots
NOMINAL — Reactor core
CAUTION — Fuel reserves
ALERT   — Hull breach sector 7
OFFLINE — Torpedo bay (N/A)

Alert Banners

Yellow border differentiates warnings from standard red controls (Rocinante pattern). Large typography but proportionate — the showrunner specifically rejected giant screen-filling alert boxes.

Alert styles
⚠ PROXIMITY WARNING — CONTACT BEARING 227 MARK 12 — 4,200 KM
⬤ COLLISION ALERT — EVASIVE ACTION REQUIRED

Data Tables

  • Thin horizontal rules between rows — no vertical lines
  • Header row: ALL CAPS, faction color, smaller than content
  • Subtle alternating row opacity — not heavy striping
  • Active row: left-edge highlight bar in faction primary

Tracking & Navigation Displays

Visual Elements

ElementStyleImplementation
Background gridFaint reference lines for distance#111 lines on #0A0A0A, ~10% opacity
Orbit pathsThin arcs showing trajectories1px, faction primary at 20% opacity
Object markersSmall diamonds/circles with labelsColor-coded: green=friendly, red=hostile, yellow=unknown
Velocity vectorsLines extending from objectsDirection + speed, proportional length
Distance readoutsNumeric near each objectMonospaced, updating in real-time
Selected targetHighlighted border, expanded infoBright border + data popout panel
Scale indicatorCorner referencee.g. "1 GRID = 500 KM"
Design note: 2D tracking screens use orbit lines and grid overlays to suggest three-dimensional space on a flat display. The show reserves true 3D for holographic projections only.

Onscreen Keyboards & Touch Interfaces

The show acknowledges that space operators may be wearing heavy industrial gloves. Interfaces designed for this have larger touch targets and may switch to more tactile inputs.

Keyboard Implementation Rules

PropertySpecification
Minimum key target48px × 48px — larger in gloved contexts
Key styleBordered rectangle, transparent fill, faction-colored border
Key gapMinimal (2–4px) — tighter than consumer keyboards
Press feedbackKey brightens or inverts momentarily (150ms)
Contextual layoutShow only what's needed — no full QWERTY when entering a 4-digit code
Physical hybridCritical functions retain physical buttons alongside touch
Kiosk variation: Ceres station kiosks feel different from Rocinante bridge controls. Personality varies by location — a port authority terminal has a different texture than a military fire control panel, even within the same faction.
08

Animation & Motion

Functional transitions only. No gratuitous animation. Motion should communicate state changes, not entertain.

TypeTimingUsage
State changes150–250msButton press, panel swap, mode toggle
Blinking indicators800ms–1.2s cycleAlert states, awaiting acknowledgment
Data updatesContinuous / real-timeNumbers count, positions update, bars fill
Terminal scrollReadable paceNot cinematic rush — operator must be able to read
Boot sequenceExtended (dramatic)Text scrolling, system checks, subsystem init
Scan linesSlow sweepBelter/older displays for retro character

Panel Component

.ex-panel {
  border:     1px solid var(--ex-border);
  background: var(--ex-panel);
  padding:    16px 20px;
}

.ex-panel-header {
  font-family:    'Overpass Mono', monospace;
  text-transform: uppercase;
  letter-spacing: 0.15em;
  font-size:      10px;
  color:          rgba(204, 0, 0, 0.7);

Button Component

.ex-btn {
  border:         1px solid var(--ex-border);
  background:     transparent;
  color:          #BBBBBB;
  text-transform: uppercase;
  letter-spacing: 0.1em;
  font-family:    'Overpass Mono', monospace;
  font-size:      11px;
  padding:        8px 22px;
  transition:     all 0.15s;
  border-radius:  0;    /* never rounded */
}
.ex-btn:hover {
  border-color: var(--ex-primary-bright);
  background:   rgba(204, 0, 0, 0.08);
  color:        #FFFFFF;
}
